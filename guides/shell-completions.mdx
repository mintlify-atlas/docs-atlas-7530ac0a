---
title: "Shell completions"
description: "Enable tab completion for xcodes commands, subcommands, and version arguments."
---

xcodes supports shell completion via [swift-argument-parser](https://github.com/apple/swift-argument-parser). Once installed, pressing <kbd>Tab</kbd> completes:

- Command and subcommand names (e.g. `xcodes ins<Tab>` → `xcodes install`)
- Xcode version numbers drawn from the available version list
- Installed Xcode versions for commands like `select` and `uninstall`

## Installation

Choose the instructions for your shell.

<Tabs>
  <Tab title="Zsh (oh-my-zsh)">
    ```bash
    mkdir ~/.oh-my-zsh/completions
    xcodes --generate-completion-script > ~/.oh-my-zsh/completions/_xcodes
    ```
  </Tab>
  <Tab title="Zsh (no oh-my-zsh)">
    ```bash
    xcodes --generate-completion-script > /usr/local/share/zsh/site-functions/_xcodes
    ```
  </Tab>
  <Tab title="Bash">
    ```bash
    xcodes --generate-completion-script > /usr/local/etc/bash_completion.d/xcodes
    ```
  </Tab>
  <Tab title="Fish">
    ```bash
    xcodes --generate-completion-script > ~/.config/fish/completions/xcodes.fish
    ```
  </Tab>
</Tabs>

<Note>
After installing the completion script, start a new terminal session or source your shell configuration file (e.g. `source ~/.zshrc`) for completions to take effect.
</Note>

## Further reading

The completions are generated by swift-argument-parser. For details on how completion scripts work, supported shells, and advanced configuration, see the [swift-argument-parser completion script documentation](https://github.com/apple/swift-argument-parser/blob/main/Sources/ArgumentParser/Documentation.docc/Articles/InstallingCompletionScripts.md).
