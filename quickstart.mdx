## Prerequisites

- macOS 10.15 or later
- An Apple ID — required to authenticate with Apple's developer portal and download Xcode

## Walkthrough

<Steps>
  <Step title="Install xcodes">
    If you have not installed xcodes yet, the quickest way is via Homebrew:

    ```bash
    brew install xcodesorg/made/xcodes
    ```

    For other installation methods, see the [installation guide](/installation).
  </Step>

  <Step title="List available versions">
    Fetch and display all Xcode versions available to install:

    ```bash
    xcodes list
    ```

    xcodes retrieves the list from [Xcode Releases](https://xcodereleases.com) by default. Installed versions are marked in the output.
  </Step>

  <Step title="Install a specific version">
    Install a version by passing its version number:

    ```bash
    xcodes install 16.2
    ```

    On first run, you will be prompted for your Apple ID username and password. xcodes stores the credentials securely in your macOS Keychain for future use.

    xcodes will then download, unarchive, move, and verify the Xcode installation automatically:

    ```
    (1/6) Downloading Xcode 11.2.0: 100%
    (2/6) Unarchiving Xcode (This can take a while)
    (3/6) Moving Xcode to /Applications/Xcode-11.2.0.app
    (4/6) Moving Xcode archive Xcode-11.2.0.xip to the Trash
    (5/6) Checking security assessment and code signing
    (6/6) Finishing installation
    ```

    You will be prompted for your macOS user password to complete the final installation step, which requires superuser privileges.

    <Tip>
      Install [aria2](https://aria2.github.io) (`brew install aria2`) to speed up downloads 3–5x. xcodes will use it automatically when it is available on your `PATH`.
    </Tip>
  </Step>

  <Step title="Switch the active Xcode version">
    After installation, make a version the active Xcode:

    ```bash
    xcodes select 16.2
    ```

    `xcodes select` calls `xcode-select` under the hood to update the system-wide active developer directory.
  </Step>

  <Step title="Verify the active version">
    Confirm that the correct version is now selected:

    ```bash
    xcode-select -p
    ```

    The output should show the path to the Xcode version you just selected, for example:

    ```
    /Applications/Xcode-16.2.app/Contents/Developer
    ```
  </Step>
</Steps>

## Install the latest version

To always install the most recent stable release without looking up the version number:

```bash
xcodes install --latest
```

To install the latest prerelease (beta, RC, or GM seed):

```bash
xcodes install --latest-prerelease
```

## Pin a version with .xcode-version

Add a `.xcode-version` file to the root of your repository to declare the exact Xcode version your project requires:

```txt .xcode-version
16.2
```

Running `xcodes select` from a directory that contains a `.xcode-version` file will automatically activate the version listed in that file — no version argument needed.

<Tip>
  Committing a `.xcode-version` file ensures that every developer on your team and every CI machine uses the same Xcode version, eliminating "works on my machine" build failures caused by mismatched toolchains.
</Tip>
